# Керування сеансами - Документація

## Функціонал скасування сеансів для адміністратора

### Огляд
Адміністратори тепер можуть скасовувати сеанси, що автоматично:
- Відміняє всі бронювання на цей сеанс
- Звільняє всі заброньовані місця
- Надсилає email-повідомлення всім користувачам, які мали бронювання

### Як скасувати сеанс

1. **Перейдіть до адмін-панелі**
   - Увійдіть як адміністратор
   - Виберіть "Адмін-панель" → "Керування сеансами"

2. **Перегляньте список сеансів**
   - У таблиці відображаються всі сеанси з інформацією:
     - Назва фільму
     - Дата і час
     - Ціна
     - Кількість заброньованих місць
     - Поточний статус

3. **Скасування сеансу**
   - Натисніть кнопку "🚫 Скасувати сеанс" для потрібного сеансу
   - Підтвердіть скасування в діалоговому вікні
   - Система автоматично:
     - Змінить статус на "Скасовано"
     - Видалить всі бронювання
     - Відправить email всім користувачам

### Статуси сеансів

**🟢 Активний** - Сеанс доступний для бронювання
- Користувачі можуть бронювати місця
- Відображається в списку доступних сеансів

**🔴 Скасовано** - Сеанс скасовано
- Бронювання неможливе
- Не відображається для користувачів
- Можна побачити лише в адмін-панелі

### Email-повідомлення

При скасуванні сеансу всі користувачі з бронюваннями отримають email:
- ⚠️ Інформацію про скасування
- 📋 Деталі скасованого сеансу
- 🔗 Посилання на перегляд інших доступних сеансів

### Технічні деталі

#### База даних
```python
class Session(db.Model):
    status = db.Column(db.String(20), default='active')  # active, cancelled
```

#### API методи
```python
# Перевірка статусу
session.is_cancelled()

# Кількість заброньованих місць
session.booked_seats_count()

# Кількість доступних місць
session.available_seats_count()
```

### Оновлення бази даних

Якщо ви вже маєте існуючу базу даних, запустіть скрипт оновлення:

```bash
python update_db_session_status.py
```

Це додасть поле `status` до всіх існуючих сеансів і встановить значення 'active'.

### Важливі зауваження

⚠️ **Скасування сеансу незворотне** - після скасування сеанс не можна активувати знову

⚠️ **Email-повідомлення** - переконайтеся що налаштовано SMTP для відправки email

💡 **Рекомендація** - скасовуйте сеанси лише у виняткових випадках (технічні проблеми, форс-мажор)

### Типові сценарії використання

**1. Технічна несправність**
```
Проблема: Збій обладнання в залі
Рішення: Скасувати сеанс, повідомити глядачів
```

**2. Відміна показу**
```
Проблема: Фільм відкликано з прокату
Рішення: Скасувати всі майбутні сеанси
```

**3. Форс-мажор**
```
Проблема: Надзвичайна ситуація
Рішення: Скасувати сеанси на певний період
```

### Моніторинг

В адмін-панелі відображається:
- Загальна кількість сеансів
- Кількість активних/скасованих
- Кількість заброньованих місць на кожний сеанс

### Приклад workflow

```
1. Адмін отримує інформацію про необхідність скасування
   ↓
2. Переходить в "Керування сеансами"
   ↓
3. Знаходить потрібний сеанс
   ↓
4. Натискає "Скасувати сеанс"
   ↓
5. Підтверджує в діалозі
   ↓
6. Система:
   - Скасовує бронювання
   - Відправляє email
   - Оновлює статус
   ↓
7. Глядачі отримують повідомлення
```

### Підтримка

Якщо виникли питання або проблеми:
- Перевірте логи додатку
- Переконайтеся що SMTP налаштовано
- Перевірте що поле `status` додано до бази даних
