# Функціонал повідомлень про нові сеанси обраних фільмів

## Опис

Коли користувач додає фільм до обраних, він автоматично отримуватиме email-повідомлення про нові сеанси цього фільму.

## Основні особливості

### 1. Пакетні повідомлення раз на день
- Система збирає всі сеанси створені протягом дня
- Раз на день (рекомендовано о 20:00-21:00) надсилає зведені повідомлення
- **Одне повідомлення на фільм на день**, навіть якщо сеанси на різні дати

### 2. Приклад роботи
**Адмін створює сеанси:**
- 10:00 - Сеанс "Опенгеймер" на 15 березня 18:00
- 11:00 - Сеанс "Опенгеймер" на 15 березня 21:00
- 14:00 - Сеанс "Опенгеймер" на 16 березня 19:00
- 15:00 - Сеанс "Опенгеймер" на 17 березня 20:00

**Користувач отримає о 21:00:**
- ✉️ **Одне** повідомлення з усіма сеансами:
  - 📅 15 березня: 18:00, 21:00
  - 📅 16 березня: 19:00
  - 📅 17 березня: 20:00

### 3. Захист від дублювання
- Система зберігає записи про надіслані повідомлення в таблиці `session_notification`
- Перевіряється комбінація `film_id` + `notification_date` (дата розсилки)
- Повторні повідомлення для того ж фільму в той же день не надсилаються

## Технічна реалізація

### Моделі

**Session** - додано поле `created_at`:
- Зберігає час створення сеансу
- Використовується для фільтрації "нових сьогодні" сеансів

**SessionNotification** - таблиця для відстеження надісланих повідомлень:
- `film_id` - ID фільму
- `notification_date` - дата коли надіслали повідомлення (YYYY-MM-DD)
- `notified_at` - точний час надсилання
- Унікальне обмеження на пару (film_id, notification_date)

### Скрипти

**send_daily_notifications.py** - головний скрипт щоденної розсилки:
1. Шукає всі сеанси створені сьогодні (`created_at` за сьогодні)
2. Групує їх по `film_id`
3. Для кожного фільму:
   - Перевіряє чи вже надсилали повідомлення сьогодні
   - Знаходить користувачів з фільмом в обраних
   - Групує сеанси по датах
   - Надсилає email з усіма датами та сеансами
   - Зберігає запис про надіслане повідомлення

### Email шаблони

- `templates/emails/new_sessions_batch_notification.html` - HTML версія
- `templates/emails/new_sessions_batch_notification.txt` - текстова версія

Показують сеанси згруповані по датах у зручному форматі.

## Використання

### Для користувачів

1. Додайте фільм до обраних (кнопка "❤️ Додати в обрані")
2. Щовечора (наприклад о 21:00) отримаєте зведення про нові сеанси
3. В листі будуть всі сеанси згруповані по датах
4. Кнопка "Переглянути і забронювати" веде на сторінку фільму

### Для адміністраторів

1. Створюйте сеанси як завжди (через форму або календар)
2. Система **не надсилає** повідомлення відразу
3. Раз на день (за розкладом) скрипт надішле зведення всім зацікавленим
4. Можна створювати сеанси на тиждень вперед - користувачі отримають одне повідомлення

## Налаштування щоденного запуску

### Windows Task Scheduler

1. Відкрийте Task Scheduler
2. Створіть нове завдання:
   - **Тригер**: Щодня о 21:00
   - **Дія**: Запуск програми
   - **Програма**: `python.exe`
   - **Аргументи**: `d:\Кузь\cinema_app\send_daily_notifications.py`
   - **Початкова папка**: `d:\Кузь\cinema_app`

### Linux/Mac Cron

Додайте в crontab:
```bash
# Щодня о 21:00
0 21 * * * cd /path/to/cinema_app && /path/to/python send_daily_notifications.py >> /var/log/cinema_notifications.log 2>&1
```

### Ручний запуск (для тестування)

```bash
cd cinema_app
python send_daily_notifications.py
```

## Оновлення бази даних

Для застосування змін виконайте:

```bash
cd cinema_app

# Додати поле created_at до Session
python update_db_session_created_at.py

# Оновити структуру SessionNotification
python update_db_notifications.py
```

## Переваги пакетного підходу

✅ **Немає спаму** - один лист на день замість багатьох  
✅ **Зручність** - вся інформація в одному місці  
✅ **Гнучкість для адміна** - можна планувати розклад на тиждень  
✅ **Контроль часу** - повідомлення приходять у зручний час

- Помилки з трейсами
